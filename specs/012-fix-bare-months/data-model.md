# Data Model — Парсинг bare-months

**Date**: 2026-04-17
**Feature**: [spec.md](./spec.md)

Фича локальна, новых сущностей в предметной области не вводит. Ниже описана модель данных на уровне входа/выхода функции парсинга — достаточно для тестирования и верификации.

## Entities

### 1. `Question` (входной текст)

**Описание**: свободный текст пользователя на русском языке.

| Поле | Тип | Описание | Ограничения |
|---|---|---|---|
| text | `str` | Исходный вопрос | UTF-8, произвольная длина; в функции приводится к `lower()` |

Пример: `"выручка март"`, `"сравни март и апрель"`, `"отчёт за месяц март"`.

### 2. `DateRange` (выходная структура)

**Описание**: пара дат, передаваемая в хранимые процедуры MSSQL как `@DateFrom` / `@DateTo`.

| Поле | Тип | Описание | Ограничения |
|---|---|---|---|
| DateFrom | `datetime.date` | Начало периода (включительно) | ≤ `DateTo` |
| DateTo | `datetime.date` | Конец периода (включительно) | ≥ `DateFrom` |

Тип возврата: `dict[str, date]` с фиксированными ключами `"DateFrom"` и `"DateTo"` (существующий контракт).

### 3. `MonthStem` (внутреннее, словарь `_MONTH_MAP`)

**Описание**: соответствие префикса названия месяца и его номера. Не меняется в рамках фичи.

| Стем | Номер |
|---|---|
| январ | 1 |
| феврал | 2 |
| март | 3 |
| апрел | 4 |
| ма(й/я) | 5 |
| июн | 6 |
| июл | 7 |
| август | 8 |
| сентябр | 9 |
| октябр | 10 |
| ноябр | 11 |
| декабр | 12 |

## State Transitions

Функция `extract_date_params` — чистая (stateless). Состояний нет.

## Validation Rules

- Если год не указан в строке, используется `date.today().year`.
- Если месяц распознан, `DateFrom = первый день месяца`, `DateTo = последний день месяца` (с учётом 28/29/30/31 и високосных лет).
- Если ничего не распознано — fallback к `last 30 days` от `date.today()`.
- Все формы нечувствительны к регистру (`question.lower()`).

## Data Flow

```text
Question(text)
   │
   ▼
extract_date_params(text)
   │  1. _RE_MONTH        ("за <месяц>")
   │  2. _RE_RANGE        ("с X по Y <месяц>")
   │  3. "вчера" / "сегодн"
   │  4. _RE_MONTH_BARE    ← новая ветка (bare-month)
   │  5. "недел"/"месяц"/"квартал"/"полугод"/"год"
   │  6. default last-30-days
   ▼
DateRange(DateFrom, DateTo) → @DateFrom, @DateTo → MSSQL stored procedure
```

## Invariants

- Сигнатура `extract_date_params(question: str) -> dict[str, date]` не меняется.
- Ключи возвращаемого словаря `{"DateFrom", "DateTo"}` не меняются.
- `has_period_hint(question: str) -> bool` возвращает тот же результат для уже поддерживаемых входов.
