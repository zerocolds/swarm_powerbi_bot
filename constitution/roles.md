# Роли ревьюеров

На каждой стадии gate-агент выбирает подмножество ролей, релевантных
артефакту. Минимум одна внутренняя и одна внешняя роль на критичных
стадиях (`implement`, `tests`, `commit`).

## Внутренние ревьюеры (Codex)

| role                   | что проверяет                                          |
|------------------------|--------------------------------------------------------|
| `architecture`         | соответствие плану, границы модулей, связанность       |
| `security`             | OWASP top-10, секреты, авторизация, инъекции          |
| `code-quality`         | читаемость, dead code, соблюдение стиля               |
| `test-engineer`        | полнота покрытия, качество ассертов, отсутствие flaky |
| `performance`          | горячие пути, аллокации, N+1                           |

## Внешние ревьюеры (Claude, другой экземпляр)

| role               | что проверяет                                             |
|--------------------|-----------------------------------------------------------|
| `external-spec`    | соответствие артефакта спеке и конституции               |
| `external-tests`   | адекватность тестов бизнес-сценариям                      |
| `external-bugs`    | поиск багов, которые пропустили внутренние ревьюеры      |

Каждая роль возвращает структуру:

```json
{
  "verdict": "approve" | "request-changes" | "block",
  "confidence": 0.0,
  "findings": [{"severity": "info|minor|major|blocker", "message": "..."}],
  "questions_for_human": ["..."]
}
```

Gate-агент агрегирует это в единое решение по правилу раздела ΙΙΙ
конституции.
